## Key Technology Stack

### Frontend
- **React 18** with TypeScript
- **Vite** for fast builds
- **Redux Toolkit** for state management
- **TailwindCSS** for styling
- **Socket.io-client** for real-time communication
- **Monaco Editor** for code editing
- **React Router** for navigation

### Backend
- **Node.js** with Express
- **TypeScript**
- **PostgreSQL** for main database
- **Redis** for caching & queues
- **Socket.io** for WebSocket
- **Prisma** ORM
- **Bull** for job queues
- **JWT** for authentication

### AI Engine
- **OpenAI SDK** (GPT-4)
- **Anthropic SDK** (Claude)
- **Google Generative AI** (Gemini)
- **LangChain** for agent orchestration
- **Vector DB** (Pinecone/Weaviate) for embeddings
- **BullMQ** for async tasks

### DevOps
- **Docker** & Docker Compose
- **Kubernetes** for orchestration
- **GitHub Actions** for CI/CD
- **Terraform** for infrastructure
- **Nginx** as reverse proxy

## Core Features Implementation

### 1. Multi-Agent System
- Product Manager Agent (requirement analysis)
- Architect Agent (system design)
- Engineer Agent (code generation)
- Data Analyst Agent (data analysis)
- Orchestrator (agent coordination)

### 2. Code Generation
- Template-based generation
- AST manipulation
- Multiple framework support
- Real-time preview

### 3. Project Management
- Project scaffolding
- File tree management
- Version control integration
- Deployment pipeline

### 4. Real-time Collaboration
- WebSocket communication
- Live code editing
- Agent status updates
- Chat interface

### 5. Authentication & Authorization
- JWT-based auth
- Role-based access control
- OAuth integration
- API key management

## Getting Started

```bash
# Install dependencies
pnpm install

# Setup environment
cp .env.example .env

# Start development
pnpm dev

# Build for production
pnpm build

# Run tests
pnpm test
```

## Agent Communication Flow

```
User Input
    ↓
Orchestrator Agent
    ↓
├─→ Product Manager → Requirement Analysis
├─→ Architect → System Design
├─→ Engineer → Code Generation
└─→ Data Analyst → Data Processing
    ↓
Response Aggregation
    ↓
User Output
```

## Database Schema

- **users**: User accounts & profiles
- **projects**: Project metadata
- **agents**: Agent configurations
- **messages**: Chat messages
- **files**: Generated files
- **deployments**: Deployment history
- **workflows**: Workflow definitions

This structure provides a complete, scalable foundation for building an AI-powered web development platform like MGX.dev!
